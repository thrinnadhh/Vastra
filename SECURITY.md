# Security Policy

## Reporting a vulnerability

Do not disclose suspected vulnerabilities, credentials, private media URLs, customer data, or
reproduction tokens in a public GitHub issue.

Report the issue privately to the repository owner through GitHub. Include:

- the affected component and commit;
- impact and prerequisites;
- minimal reproduction steps;
- whether any secret or customer data may have been exposed; and
- a safe remediation suggestion, if known.

Do not test against production accounts or data without explicit authorization. Use local or
dedicated test environments and redact all tokens, OTPs, personal data, and signed URLs.

## Supported versions

Security fixes are applied to the current `main` branch during MVP development. Released pilot
versions will receive an explicit support table when versioned releases begin.
