# Global Codex Working Agreements

- Prefer `pnpm` for JavaScript and TypeScript repositories.
- Inspect repository instructions before planning.
- Work on one focused task at a time.
- Ask before adding a new production dependency.
- Preserve uncommitted user changes.
- Never expose or print secrets.
- Run relevant tests after changes.
- Do not claim success when commands fail or were not run.
- Prefer reversible changes and small diffs.
- Create or recommend a Git checkpoint before risky work.
- Explain unresolved uncertainty instead of guessing.
