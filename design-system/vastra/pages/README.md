# Vastra page-specific design overrides

Add a page file here only when a screen intentionally diverges from `../MASTER.md`.

Every override must document:

- page or flow name;
- affected platform and breakpoint;
- exact master rule being overridden;
- user or operational reason;
- accessibility impact;
- design owner;
- expiry or review condition where applicable.

An override must not weaken privacy, contrast, touch-target, safe-area, reduced-motion, keyboard, or destructive-action requirements.
